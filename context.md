# Context — Hand Dexterity Assessment

## Hackathon Track

**AI Vibe Coding** — ตรวจความคล่องแคล่วมือผ่านวิดีโอ

เป้าหมาย: วิเคราะห์ว่ามือข้างไหนถนัดจริง โดยดูจากความเร็ว ความแม่นยำ และคุณภาพการเคลื่อนไหว และแยกจากกรณี **Learned Non-Use** (ผู้ป่วยเลิกใช้แขนข้างที่อ่อนแรงจน "ลืม" วิธีใช้) และช่วยฝึกการเคลื่อนไหวของมือ

## Domain Knowledge

### Learned Non-Use

ภาวะที่ผู้ป่วยหลีกเลี่ยงการใช้แขนข้างที่อ่อแรงหลัง stroke หรือบาดเจ็บ แล้วใช้ข้างที่แข็งแรงแทนจนแขนที่อ่อนแรงเสื่อมลงเรื่อยๆ ระบบนี้มองหา pattern:

- มือหนึ่งข้างมีคะแนน speed/accuracy/quality สูงกว่าอีกข้างอย่างชัดเจน
- มือที่อ่อนแรงหรือมีภาวะ learned Non -used ยังมี success rate บางแต่ quality ต่ำ (เคลื่อนไหวกระตุก)และพยายามใช้มืออีกข้างที่ผิด 

### มือถนัด (Hand Dominance)

มือที่ใช้เป็นหลักในชีวิตประจำวัน มักมี:

- reaction time สั้นกว่า
- แตะเป้าหมายแม่นยำกว่า
- เส้นทางการเคลื่อนไหว smooth กว่า (jerk ต่ำ, path efficiency สูง)

## เกณฑ์การประเมิน (AI Metrics)

| เมตริก | คำอธิบาย (สำหรับผู้ใช้) | การคำนวณ |
|--------|-------------------------|----------|
| **ความเร็ว (Speed)** | มือเคลื่อนไปถึงเป้าหมายเร็วแค่ไหน | reaction time + path length / duration |
| **ความแม่นยำ (Accuracy)** | แตะตรงกลางเป้าหมายแค่ไหน | 1 − (ระยะจากจุดกลาง / รัศมีเป้า) |
| **คุณภาพการเคลื่อนไหว (Quality)** | การเคลื่อนไหวนุ่มนวลหรือกระตุก | jerk (อนุพันธ์อันดับ 3) + path efficiency |

## Key Focus — UX สำหรับผู้สูงอายุ

### Accessible UX

- ฟอนต์ขั้นต่ำ 24px
- Contrast ratio ≥ 7:1 (WCAG AAA)
- ปุ่มขั้นต่ำ 64×64 px
- Focus ring สีส้ม 4px
- `aria-live` สำหรับคะแนนและ feedback
- รองรับ keyboard navigation

### Easy Interface

- Flow 3 ขั้น: ยินดีต้อนรับ → เล่นเกม → ดูผล
- ไม่มี countdown กดดัน
- Progress bar แสดงความคืบหน้า
- คำแนะนำภาษาไทยสั้นๆ ทุกขั้น
- สีมือทั้งสองข้างเปลี่ยนไปตามแต่ละรอบ

### Empathetic Design

- ข้อความสำเร็จ: "เก่งมาก!", "ดีมากเลย!", "ทำได้ดีมาก!"
- ข้อความพลาด: "ไม่เป็นไร ลองอีกครั้งนะ" 
- Web Speech API อ่านคำสั่งภาษาไทย (ถ้า browser รองรับ)
- Disclaimer ชัดเจนว่าไม่ใช่การวินิจฉัยทางการแพทย์

## ข้อความ UI

### หน้ายินดีต้อนรับ

- หัวข้อ: "ยินดีต้อนรับ"
- คำอธิบาย: "เกมนี้ช่วยประเมินการเคลื่อนไหวของมือทั้งสองข้าง และฝึกให้ใช้มือทั้ง 2 ข้าง"
- ปุ่ม: "เริ่มเล่น"

### หน้าเกม

- คำสั่งหลัก: "ใส่สี"
- Calibration: "ยกมือทั้งสองข้างให้กล้องเห็น"
- รอบเล่น: มีสิบสองรอบแต่ละรอบจะสุ่มสีมือ ทั้งสองข้างให้นำมือทั้งสองข้างไปใส่สีให้ถูก 

### Template ผลลัพธ์ (4 classes)

**right_dominant:**
> มือขวาของคุณเคลื่อนไหวได้คล่อง แม่นยำ และนุ่มนวลกว่ามือซ้าย ซึ่งสอดคล้องกับการใช้มือขวาเป็นหลักในชีวิตประจำวัน

**left_dominant:**
> มือซ้ายของคุณเคลื่อนไหวได้คล่อง แม่นยำ และนุ่มนวลกว่ามือขวา ซึ่งสอดคล้องกับการใช้มือซ้ายเป็นหลักในชีวิตประจำวัน

**learned_non_use_left:**
> คุณใช้มือขวาได้ดีกว่ามือซ้ายอย่างชัดเจน อาจมีแนวโน้มของ Learned Non-Use ที่มือซ้าย — แนะนำให้ฝึกใช้มือซ้ายอย่างนุ่มนวลและสม่ำเสมอ

**learned_non_use_right:**
> คุณใช้มือซ้ายได้ดีกว่ามือขวาอย่างชัดเจน อาจมีแนวโน้มของ Learned Non-Use ที่มือขวา — แนะนำให้ฝึกใช้มือขวาอย่างนุ่มนวลและสม่ำเสมอ

## เกม Mechanics

- มือซ้าย = เริ่มต้น สีแดง และเปลี่ยนสีเมื่อสำเร็จรอบ, มือขวา = เริ่มต้น สีน้ำเงิน และเปลี่ยนสีเมื่อสำเร็จรอบ
- เป้าหมาย 3 วง: เริ่มต้นที่ สีแดง,สีน้ำเงิน ,สีม่วง สีจะเปลี่ยนไปเรื่อยๆเมื่อชนะ ไม่ซ้ำกันแต่ละด่าน  — overlay ด้านบนของภาพกล้อง
- มือทั้งสองข้างจะมีสีและใส่ได้ทั้งสองข้าง โดยแต่ละรอบจะมีให้ใส่ทั้งสองข้าง เช่น มือซ้าย สีแดง ก็ใส่สีแดง มีขวา สีเขียว ก็ใส่สีเขียว
- มีท้้งหมด 12 รอบ 
- เมื่อใส่สีผิดจะไม่ได้คะแนน 
- เมื่อจบเกมจะเด้งป้อบอัพแสดงข้อความยินดีด้วยและแสดงคะแนน
- Collision radius: 120px
- MediaPipe landmark index 9 (middle finger MCP)
